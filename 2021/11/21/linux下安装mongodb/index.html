linux下安装mongodb

发表于2021-11-21|更新于2021-11-21|mongodb
|字数总计:888|阅读时长:3分钟|阅读量:

安装

下载地址

  • 下载地址：https://www.mongodb.com/try/download/community
  • 根据自己的系统选择相应的平台和安装包的格式。

连接到服务器的终端

如果自己的系统是linux系统就不需要，现在我的系统是window系统，这里我就采用的是自己的服务器来进行安装

  • 这里我选择Xshell进行连接（xshell6是免费的）
  • 这里我选择XFTP来进行文件的上传（把我们下载好的安装包上传到服务器上）
  • 服务器的连接和文件的上传这里我就不再讲述。需要的小伙伴可以百度搜索。

解压文件

找到mongodb的压缩包（.tgz）的路径，使用以下命令进行解压

  • 解压
  • 将解压后的文件移动到指定文件的位置，当然文件的位置可以自己选择，这里我们选择在usr/local/mongodb文件目录下
  • 创建数据库文件和日志文件:data文件夹是数据库存放的位置，logs是日志文件
    • /usr/local/mongodb/bin/mongod：mongodb可执行文件；
    • –dbpath=xxx：指定数据库存放的位置；
    • –logpath=xxx： 日志文件存放的位置；
    • –logappend：日志启动追加操作；
    • –auth：开启认证登陆；
    • -–port=27017：指定端口号
  • 设置开机自动启动服务
  • 启动mongodb服务
  • 验证
    在mongodb安装的bin目录下输入以下命令;若出现版本号说明安装成功

配置

每次的启动显得特别的麻烦，所有我们创建一个配置文件

  • 在mongodb的安装目录下创建一个mongo.conf配置文件
  • 使用vim来编辑配置文件
  • 启动mongodb

mongodb环境变量的配置

  • 在/etc.profle文件中进行配置
  • 在文件之后添加如下代码：
  • export PATH=$PATH:mongodb安装的文件地址/bin
  • 重新加载配置source /etc/profile
  • 启动指令mongod -f mongo.conf的文件地址

mongodb认证设置

  • 使用mongo实现客户端链接
  • user admin使用admin数据库
  • 创建一个用户db.createUser({user:"admin",pwd:"password",roles:["root"]})
  • 认证登录db.auth("admin", "password")

使用Robo 3T进行连接

  • 下载地址：https://robomongo.org/download/
  • 在连接的时候，需要输入认证的账号和密码以及连接的mongodb服务器的地址

备注

在使用的过程中，我们可能会遇到bug

about to fork child process, waiting until server is ready for connections.
forked process: 2340
ERROR: child process failed, exited with error number 1

解决

  • cd 到配置的数据库文件中
  • rm mongodb.lock文件
  • mongod --repair --dbpath=/usr/local/mongodb/data修复文件
