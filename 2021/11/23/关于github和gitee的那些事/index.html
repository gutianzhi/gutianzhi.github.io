关于github和gitee的那些事

发表于2021-11-23|更新于2021-11-24|BUG
|字数总计:834|阅读时长:2分钟|阅读量:

事情的原由

最近在使用Hexo框架来搭建自己的博客，由于开始想把自己的项目部署到gitee上面。但是想了想不如把项目同时也部署到github上面，本想很快解决，但是中途输到了很多Bug。但是在自己快要崩溃的边缘，抱着再试一试的态度，终于迎来了胜利的曙光。记录一下学习的过程。

github

访问拦路虎

由于github的服务器是在国外，国内访问显然是比较慢的，甚至有时候会崩溃。这样最容易搞人心态。遇到问题不要慌，找到度娘棒棒忙。

访问镜像

  • https://github.com.cnpmjs.org
  • https://hub.fastgit.org

这种方式我也尝试过，好像并没有多大的作用。访问速度并没有我想象中的那样快。

修改本地的hosts文件

  • 访问地址：http://tool.chinaz.com/dns/ 开查询github的地址
  • 将查询到的地址添加到系统的hosts文件中。
  • hosts文件的地址：C:\Windows\System32\drivers\etc\hosts
  • 将查询到的ip添加到hosts文件中的格式：ip地址 访问的地址；例如 xxx.xxx.xxx.xxx github.com
  • 如果觉得比较麻烦，可以在百度上进行搜索，直接Copy。
  • 如果出现无法修改hosts文件，我们需要修改文件的权限：右击->hosts->属性->安全->编辑->点击Users->在Users的权限“写入”后面打勾。
  • Win+R输入cmd——>ipconfig /flushdns刷新DNS。

大致的思路就是通过查询对应地址的IP地址，修改hosts文件，主动建立域名与 IP 的映射关系，访问到这些域名时直接使用 Hosts 指定的 IP，绕过 DNS 解析。

下载VPN插件

在以上两种方法中折腾了半天，还是无果。于是自己就换了一种思路，直接使用VPN翻墙。但是想法是好的，搜索插件就给我整了半天，唉~~~。

  • 推荐浏览器：Microsoft Edge
  • 在浏览器中的扩展插件中搜索VPN Professional。该插件是我试了无数坑才找出来的。

修改github

以上解决了网址的访问题，接下来就解决我们hexo部署的问题。

部署Hexo项目在Github上

  • 安装Git
  • 创建自己的仓库。命名为：Github账号名.github.io
  • 在Hexo中配置repo的地址：可以采用Https或者SSH，个人比较喜欢采用SSH，因为Https在每次的提交中要输入自己的密码和账户。
  • 如果不存在，就输入以下的指令。
  • 打开对应的目录文件.ssh\id_rsa.pub
  • github主页，进入个人设置 -> SSH and GPG keys -> New SSH key配置公钥。
  • 测试：
  • 可能出现的问题：Are you sure you want to continue connecting (yes/no)?:输入yes即可。
